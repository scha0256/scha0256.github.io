Navbars use the class .navbar along with . navbar-expand in order to cancel out the colour scheme classses bootstrap has in place.

A card is designed to be a visible container on your site to show content. It can include header, footees, background colours, and display options.
Cards have no fixed width so they will fill the full width of the parent element without added elements. Cards supports multiple content including images, text, list groups, links, etc.

The .card-body element is used for padding , it is the building block of every card.
The .card-title is the same as a "h1" tag and .card-link is the same as using a "a" tag.

Grid in Bootstrap is a series of containers, rows and columns. Using flexbox bootstrap creates a format that is already fully responsive.

  • Grid supports 6 responsive breakpoints which are based off of min-width media queries. This meaning you can control each column with sizing.
  • Use the .container to achieve a responsive width and .container-fluid for a width that is 100% across all viewports.
  • Every column has a horizontal padding element caled a (gutter) which controls the space between them. Rows also use modifier class to uniformly change column sizes and gutter classes.


Bootstrap's 6 default brakpoints include:

  • Extra small (xs)
  • Small (sm)
  • Medium (md)
  • Large (lg)
  • Extra large (xl)
  • Extra extra large (xxl)

Media Queries


Bootstrap is set to design mobile first. Their media queries are built on breakpints that are mostly based on the minimum viewport width to allw changes.
The following media queries are the common breakpoints for bootstraps layout.

Small devices (landscape phones, 576px and up)
@media (min-width: 576px) { ... }

Medium devices (tablets, 768px and up)
@media (min-width: 768px) { ... }

Large devices (desktops, 992px and up)
@media (min-width: 992px) { ... }

X-Large devices (large desktops, 1200px and up)
@media (min-width: 1200px) { ... }

XX-Large devices (larger desktops, 1400px and up)
@media (min-width: 1400px) { ... }

Making Responsive


Although media queries are good there is more than just the defualt breakpoints you can use to make a site responsive. By adding certain classes to elements in you code your website can be more responsive.

An example of that is the navbar. At the top of the page the navbard example uses a class "collapse navbar-collapse", this class makes the menu apear like a stack when viewing in smaller browsers.
